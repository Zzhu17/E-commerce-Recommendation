# 团队安全培训计划（年度）

> 强制主题：密钥管理、钓鱼防护、最小权限开发规范。

## 1. 培训对象
- Backend/Reco/MLE/DA/SRE 全员。
- 新成员入职 30 天内完成基础培训。
- 拥有管理权限人员需追加高权限专项培训。

## 2. 课程结构

### 模块 A：密钥管理（必修）
- 禁止明文密钥入代码、配置、日志与工单。
- 正确使用 Vault/KMS/Secret Manager 注入。
- 密钥轮换策略、泄露应急与吊销流程。
- 实操：在沙箱环境完成一次“轮换 + 无停机切换”。

### 模块 B：钓鱼防护（必修）
- 常见邮件/IM 钓鱼手法与社会工程识别。
- 可疑链接/附件处置流程（上报、隔离、验证）。
- MFA、设备安全、浏览器扩展最小化。
- 演练：季度钓鱼模拟（点击率与上报率评估）。

### 模块 C：最小权限开发规范（必修）
- API 与数据库权限最小化设计。
- 禁止“全库读写”默认权限与共享账号。
- 临时授权（JIT）申请与自动回收。
- Code Review 安全检查清单（鉴权、输入校验、日志脱敏）。

## 3. 节奏与考核

- 年度必修课：1 次（2-3 小时）。
- 季度微课：每季度 1 次（30-45 分钟）。
- 演练：
  - 钓鱼模拟：每季度 1 次。
  - 密钥泄露应急演练：每半年 1 次。

考核指标：
- 必修完成率 >= 100%。
- 考核通过率 >= 90%。
- 钓鱼模拟点击率逐季下降，上报率逐季提升。

## 4. 记录与追踪

每次培训需留档：
- 参与人员名单与完成状态。
- 课件与测验结果。
- 发现问题与整改项（负责人、截止日期）。

## 5. 不合规处理

- 未按期完成必修培训：暂停生产高权限访问。
- 连续两次考核未通过：要求补训并限制变更审批权限。
