# 内部访问控制矩阵（行为数据）

> 目标：明确“谁能看原始行为数据、谁能导出、谁能删”，并落实审批与审计。

## 1. 角色定义

- **Data Platform Admin（DPA）**：数据平台运维与权限管理。
- **Security Officer（SEC）**：安全策略、审批、审计复核。
- **Recommendation Engineer（RECO）**：推荐服务开发与排障。
- **Data Analyst（DA）**：分析报表与效果评估。
- **ML Engineer（MLE）**：模型训练与特征工程。
- **SRE**：系统稳定性与应急支持。

## 2. 权限矩阵

| 操作 | DPA | SEC | RECO | DA | MLE | SRE |
|---|---|---|---|---|---|---|
| 查看去标识行为数据（`feedback_events`） | 允许 | 允许（审计目的） | 允许（最小字段） | 允许（最小字段） | 允许（最小字段） | 仅故障场景临时授权 |
| 查看原始行为数据（含可识别标识） | 最小必要允许 | 允许（审计） | 默认禁止 | 默认禁止 | 默认禁止 | 默认禁止 |
| 导出去标识数据 | 允许（需工单） | 允许（审批后） | 需审批 | 需审批 | 需审批 | 禁止 |
| 导出原始行为数据 | 需双人审批（SEC + 业务Owner） | 需双人审批 | 禁止 | 禁止 | 禁止 | 禁止 |
| 执行用户删除（`delete-user`） | 允许（需工单） | 审批/复核 | 可发起申请，不可直删 | 可发起申请，不可直删 | 可发起申请，不可直删 | 禁止 |
| 分配/回收权限 | 允许（双人复核） | 复核 | 禁止 | 禁止 | 禁止 | 禁止 |

## 3. 审批与执行规则

1. **查看原始数据**：
   - 仅在安全调查、合规审计、严重故障根因定位时允许。
   - 必须记录用途、数据范围、开始/结束时间。
2. **导出数据**：
   - 去标识数据：工单审批（用途、最小字段、留存天数、存放位置）。
   - 原始数据：双人审批 + 到期自动销毁（默认 7 天）。
3. **删除数据**：
   - 通过标准接口执行，禁止手工 SQL 直接删主业务表。
   - 删除后必须写审计记录并可追溯到工单号。

## 4. 技术控制要求

- RBAC：按角色授予数据库 schema/table/column 最小权限。
- JIT 临时授权：高敏权限默认无常驻，按需临时开通，过期自动回收。
- 审计日志：记录查询主体、语句摘要、导出条数、下载目标、审批单号。
- 防扩散：导出文件强制加密与水印标识（导出人、时间、工单号）。

## 5. 违规处理

- 未经审批访问/导出/删除，视为高风险违规；立即冻结账号并触发安全事件流程。
- 一周内完成事件复盘并输出整改清单（权限、流程、培训三类）。
