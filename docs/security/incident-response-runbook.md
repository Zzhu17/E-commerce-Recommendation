# 安全事件响应 Runbook

## 1) 触发条件与分级
- **P0（15 分钟内响应）**：凭据泄漏、明确入侵、管理员权限被滥用。
- **P1（30 分钟内响应）**：401/403 激增、admin 接口异常调用、失败登录来源聚集。
- **P2（2 小时内响应）**：单点可疑行为但未扩散。

## 2) 发现阶段（T+0 ~ T+15 分钟）
1. 在监控系统确认告警来源：`security_auth_failures_total`、`security_forbidden_total`、`security_admin_calls_total`、`security_failed_login_source_total`。
2. 以 `requestId` 串联查询 gateway/backend/model-api 日志。
3. 仅使用脱敏字段（`userId/itemId/requestBody` 已默认脱敏），禁止在排查记录中粘贴原始敏感参数。

## 3) 遏制阶段（T+15 ~ T+60 分钟）
1. **疑似泄漏**：
   - 立即轮转 JWT Secret、数据库密码、外部 Webhook 密钥。
   - 吊销高风险 token 与会话。
2. **疑似入侵**：
   - 在 gateway 封禁可疑来源 IP/UA。
   - 临时收紧 admin 接口访问（仅平台管理员 + IP 白名单）。
3. 建立事件频道：
   - 安全事件：Pager（主）+ Slack `#sec-incident` + 邮件抄送安全组。
   - 可用性事件：Pager/Slack `#ops-incident`，不混发安全频道。

## 4) 根除与恢复（T+1h ~ T+24h）
1. 清理恶意任务、回滚异常配置、修补暴露面。
2. 对受影响账号执行强制登出与凭据重置。
3. 验证恢复条件：
   - 安全类指标回落到基线。
   - 无新增高危告警连续 60 分钟。

## 5) 复盘与时限
- **24 小时内**提交初步事故报告（影响范围、时间线、根因假设、临时措施）。
- **72 小时内**提交完整 RCA（根因、长期修复、检测缺口、责任人与截止日期）。
- 所有修复项必须绑定工单与上线窗口，禁止口头跟踪。

## 6) 值班操作清单（执行即打勾）
- [ ] 告警已确认并分级。
- [ ] 负责人、沟通频道、升级路径已建立。
- [ ] requestId 追踪完成且证据脱敏。
- [ ] 完成遏制动作（封禁/轮转/下线风险入口）。
- [ ] 恢复验证通过并持续观测 60 分钟。
- [ ] 事故报告与 RCA 已按时提交。
