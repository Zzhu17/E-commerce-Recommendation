# DPIA + 威胁建模（推荐链路）

> 范围：`/api/recommendations` 与 `/api/feedback` 全链路（前端 -> 后端 -> 模型服务 -> 存储/分析）。

## 1. 处理活动概述（DPIA）

### 1.1 业务目的
- 提供个性化推荐结果。
- 基于反馈事件进行模型评估与迭代。
- 保障系统稳定性与安全可审计性。

### 1.2 涉及数据
- 直接标识/准标识：`userId`（在线态）、`user_token`（存储态）。
- 行为数据：`itemId`、`eventType`、`scene`、`ts`。
- 运维数据：`requestId`、状态码、耗时、调用路径。

### 1.3 必要性与比例性
- 必要字段：用于推荐、反馈归因与防滥用，属于服务交付最小集合。
- 可选字段：`modelVersion/extra` 默认关闭，仅在实验需要时开启。
- 数据最小化：反馈入库采用去标识 token，避免明文用户标识长期落库。

## 2. 数据流（Data Flow）

```text
[User Browser]
   | 1. /api/recommendations (userId, scene, size, requestId)
   v
[Spring Backend]
   | 2. 调用 Model API (requestId, userId, candidate context)
   v
[Model API]
   | 3. 返回 ranked items + modelVersion
   v
[Spring Backend] ----> [Access Logs / Metrics]
   | 4. 返回推荐结果给前端
   |
   | 5. /api/feedback (requestId, userId, itemId, eventType, scene, ts)
   v
[Feedback Sink]
   | 6. userId -> user_token (HMAC)
   v
[Postgres feedback_events]
   | 7. 下游进入离线评估/训练管道（最小字段集）
   v
[Warehouse / Feature / Training]
```

## 3. 威胁建模（STRIDE）

| 威胁 | 攻击面 | 风险 | 现有控制 | 加固建议 |
|---|---|---|---|---|
| S（冒充） | 伪造调用管理接口 | 越权导出/删除 | API Key/JWT、网关鉴权 | 管理接口绑定 MFA + IP allowlist |
| T（篡改） | 篡改反馈事件或模型结果 | 训练污染、推荐偏移 | 请求校验、服务端字段白名单 | 增加签名校验与重放保护 |
| R（抵赖） | 否认导出/删除行为 | 审计缺失 | 审计表记录（删除） | 导出操作补齐审计链路 |
| I（信息泄露） | 原始行为数据被误查或误导出 | 隐私泄露 | token 化、日志脱敏 | 行级/列级权限 + 水印导出 |
| D（拒绝服务） | 高并发刷接口/恶意流量 | 服务不可用 | 限流、防护过滤器 | WAF/行为验证码（按场景） |
| E（权限提升） | 低权限账号获得高权限数据访问 | 横向扩权 | 角色分离 | 季度权限复核 + JIT 临时授权 |

## 4. DPIA 风险评估结论

| 风险项 | 可能性 | 影响 | 等级 | 处置 |
|---|---|---|---|---|
| 原始行为数据泄露 | 中 | 高 | 高 | 严格 RBAC、导出审批、审计告警 |
| 反馈数据污染 | 中 | 中 | 中 | 输入校验、异常检测、回滚机制 |
| 管理接口滥用 | 低-中 | 高 | 中-高 | 双人审批、MFA、密钥轮换 |
| 过度采集 | 低 | 中 | 中 | 默认关闭可选字段、变更评审 |

结论：在“最小化采集 + 去标识化存储 + 权限隔离 + 审计复核”执行到位前提下，剩余风险可控；高风险项需持续监控并在季度评审中复核。

## 5. 必做整改项（与本次治理配套）

1. 落地访问控制矩阵（原始数据查看、导出、删除的角色边界）。
2. 对导出操作补齐审批与审计（工单号、审批人、用途、数据范围、留存位置）。
3. 建立季度权限复核与年度渗透测试闭环。
4. 将密钥管理、钓鱼防护、最小权限开发纳入年度强制培训。
